# Network Devices

## What are Network Devices?

Network devices are physical devices that help computers and other systems communicate across a network.

Each device performs a specific function to help move, manage, or secure network traffic.

---

## Why Network Devices Matter

Without network devices, computers would not be able to communicate efficiently.

Network devices help:

* Connect devices
* Forward traffic
* Provide internet access
* Improve security
* Manage network communications

---

## Hub

A hub is a Layer 1 device that sends incoming data to every connected device.

### Key Facts

* Operates at Layer 1
* Broadcasts traffic to all devices
* Rarely used in modern networks

---

## Switch

A switch is a Layer 2 device that forwards data using MAC addresses.

### Key Facts

* Operates at Layer 2
* Uses MAC addresses
* Creates separate collision domains
* Common in modern networks

---

## Router

A router connects different networks together and forwards traffic between them.

### Key Facts

* Operates at Layer 3
* Uses IP addresses
* Breaks broadcast domains
* Often serves as the default gateway

---

## Modem

A modem connects a home or business network to an Internet Service Provider (ISP).

### Key Facts

* Provides internet connectivity
* Converts signals between the ISP and local network
* Often combined with a router in home networks

---

## Wireless Access Point (AP)

A wireless access point allows wireless devices to connect to a network.

### Key Facts

* Provides Wi-Fi access
* Extends network connectivity
* Common in homes, schools, and businesses

---

## Firewall

A firewall is a security device that monitors and controls network traffic.

### Key Facts

* Filters incoming traffic
* Filters outgoing traffic
* Helps prevent unauthorized access
* Important for cybersecurity

---

## Real-World Example

Home Network:

Internet
↓
Modem
↓
Router
↓
Switch
↓
Computers, Phones, Consoles

The router provides internet access, the switch connects devices, and the access point provides Wi-Fi.

## Commands to Practice

```bash
# View local network configuration and find your default gateway
ipconfig

# View detailed network configuration, including MAC and DNS info
ipconfig /all

# Test connectivity to an external host
ping google.com

# Trace the path packets take to leave your network segment
tracert google.com

# View your local ARP table mapping IP to MAC addresses
arp -a
